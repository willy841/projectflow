#### Vendor Detail
- 正式保留三大區塊：
  1. A. 廠商主檔區
  2. B. 未付款專案區
  3. C. 合作紀錄區
- 上半段左右版型已驗收通過：
  - 左：廠商主檔區
  - 右：未付款專案區

  - 匯款資料：銀行、戶名、帳號、身分證字號
- 未付款專案區：
  - 承接已完成確認對帳且尚未付款的 `專案 × 廠商`
  - 保留勾選框、專案名稱、摘要說明、未付款金額
  - 可批次 `標記為已付款`
- 合作紀錄區：
  - 保留狀態：`執行中` / `未付款` / `已結案`
  - 移除 `已付款`
  - 卡片只保留：專案名稱、專案狀態、金額
  - 金額抓取專案按全部確認後的正式成立值
  - 展開後保留成本明細與發包明細，但不顯示來源資訊

#### Vendor Data 與上下游對齊
- Vendor Data 不承接帳務中心
- Vendor Data 的未付款與合作紀錄，只承接報價成本中完成確認對帳後正式成立的金額
- 承接單位為：`專案 × 廠商`
- 與上游 `Project Detail` 的正式對齊點：
  1. 建立完成的廠商，必須能在 `Project Detail` 被選到
  2. 在廠商資料模組中已建立的工種，也必須能在 `Project Detail` 被選到

## 2.4 款項追蹤表（已討論到可寫入 spec 的程度）
### 已定內容
- 款項追蹤表是帳務中心中 `執行中專案` 區塊下的主表
- 以專案為主體
- 上方有 `執行中專案` 總計區塊，保留：
  1. 總金額
  2. 已收款
  3. 未收款
- 總金額定義：未結案狀態下，只要已匯入報價單即成立
- 已收款定義：目前以訂金為準，對應報價成本手動輸入的訂金金額
- 未收款定義：尚未收回的剩餘款項；若訂金未收則含訂金 + 尾款，若訂金已收則只算尾款
- 款項追蹤表每列保留欄位：
  1. 專案名稱
  2. 活動日期
  3. 總金額
  4. 已收款
  5. 未收款
- 目前不額外拆訂金 / 尾款狀態欄
- 保留 `查看詳情` 作為單案導流入口

# 4. 本輪尚未正式定案的討論題目

這一題已被明確指出先前混在一起，後續必須嚴格分開：
- 輸入區 = 用來新增 / 填寫當月成本資料

目前仍未完成正式 spec。

## 4.2 人事資料：固定設定 vs 每月紀錄
已經確認：
- 加班時數與獎金是每月變動值
- 不應只放在同仁主檔
- 但目前尚未正式定義「同仁主檔」與「每月人事紀錄」的完整 UI 與資料責任

目前方向是：
- 先收口 `款項追蹤表`
