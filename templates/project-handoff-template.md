# 新 Chat 銜接 SOP / Project Handoff Template

每次專案系統建置相關工作要切換到新 chat、新 session 或新一輪接手時，都應先整理以下內容。

預設規則：當對話已形成長 context、已有重要決策、已有進度或已進入執行時，酷亞大總管應在舊對話結束前主動產出或更新 handoff md 檔，供下一輪接手使用。

## 1. 專案名稱
- 

## 2. 專案目標
- 這個系統要解決什麼問題？
- 當前主要目標是什麼？

## 3. 目前階段
- 例如：需求釐清 / 規格定義 / UI 設計 / 工程實作 / 整合 / 驗收

## 4. 本輪 scope
- 這一輪明確要做什麼
- 這一輪明確不做什麼

## 5. 已完成事項
- 已完成哪些功能、決策、文件或驗收項目

## 6. 進行中事項
- 正在進行什麼
- 由誰負責（CPO / CTO / 前端設計 Agent）

## 7. 待批准事項
- 哪些事項仍等待使用者批准
- 哪些還不能正式派工

## 8. 阻塞與風險
- 目前卡住什麼
- 有哪些產品、設計、技術或排程風險

## 9. 最新重要決策
- 最近做了哪些重要決策
- 為什麼這樣決定

## 10. 團隊分工狀態
- CPO 目前負責什麼
- CTO 目前負責什麼
- 前端設計 Agent 目前負責什麼

## 11. 下一步
- 接下來最優先的 1–3 個動作

## 12. 必讀材料
- 相關 memory
- 最新 handoff
- 團隊章程
- 必讀專案檔案 / 文件 / repo 路徑

## 13. 新 Chat 開場檢查清單
新 chat 開始時，酷亞大總管必須先做：
- 讀取 relevant memory
- 讀取團隊章程
- 讀取最新 handoff
- 確認目前階段、scope、待批准事項與下一步
- 再開始接續討論或派工

## 14. 規則
- 沒有 handoff 時，不要假裝完整承接長 context
- 找不到資料時，要明講缺口
- 涉及舊決策、偏好、人物、日期、待辦時，要主動使用 memory search
- handoff 應簡潔但足以讓新 chat 快速接手
