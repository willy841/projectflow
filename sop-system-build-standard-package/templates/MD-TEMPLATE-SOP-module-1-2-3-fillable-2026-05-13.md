# 模組 1：名詞定義表（可直接填寫版）

## 這份表要解決什麼

很多系統不是做不出來，而是大家講的是同一個詞，理解卻不一樣。

例如：
- 專案 是什麼？
- 任務 是什麼？
- 完成 和 結案 有沒有不同？
- 確認 是不是正式拍板？

如果這些詞一開始沒有講清楚，後面很容易：
- 使用者講 A
- 執行者理解成 B
- 系統做成 C

所以這份表的目的，是把核心名詞先講清楚。

---

## 填寫方式

每一列填一個名詞。

如果你覺得某個詞很常講、很重要、很容易誤會，就應該列進來。

# 模組 2：角色與權限表（可直接填寫版）

## 這份表要解決什麼

這份表是要先講清楚：
- 系統裡有哪些人
- 每種人平常在做什麼
- 他能看什麼
- 他能改什麼
- 哪些動作只有特定角色能做

如果這個沒先定，後面很容易變成：
- 按鈕不知道該給誰看
- 欄位不知道誰能改
- 確認、刪除、結案這些高風險動作沒邊界

---

每一列填一種角色。

不要只填職稱，請盡量用「他平常負責什麼」去描述。

# 模組 3：流程與狀態表（可直接填寫版）

## 這份表要解決什麼

這份表要講清楚：
- 一件事情怎麼開始
- 中間怎麼走
- 哪一刻算正式確認
- 什麼時候算完成
- 如果做錯、取消、重開，要怎麼處理

很多系統主流程看起來都差不多，
真正容易出問題的是：
- 做到一半取消怎麼辦
- 確認後能不能改
- 改了之後會影響哪裡
- 要不要重開流程

---

## 填寫方式

一個流程可以拆成多個步驟來填。

建議先填主流程，再補例外流程。

## 填寫提醒

如果你不知道怎麼填，可以先口語回答：

1. 這件事第一步是什麼？
2. 做完第一步後，下一步由誰接？
3. 哪一刻算正式確認？
4. 確認後還能不能改？
5. 如果取消、做錯、重來，怎麼處理？

把這些話先寫出來，再整理成步驟就可以。
