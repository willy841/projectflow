# MD49 - projectflow Accounting Center 營收概況 v1 spec（2026-04-08）

> 目的：定義 `projectflow` 下游階段中，Accounting Center / 帳務中心之下 `營收概況` 區塊的 v1 正式規格。
>
> 本檔承接：
> - `MD40-projectflow-closeout-module-v1-spec-2026-04-08.md`
> - `MD44-projectflow-accounting-center-operating-expense-module-v1-spec-2026-04-08.md`
>
> 本檔目前先不碰 DB、API、persistence，只定義前端功能架構、時間主體、摘要卡規格與模組對齊語意。

---

# 1. 模組定位

`營收概況` 的正式定位為：

> **用來查看某個時間範圍下，收入、成本、營運支出與最終利潤的營運摘要區**

本區只處理：
- 時間範圍切換
- 查看收入 / 成本 / 支出 / 利潤摘要值

本區不處理：
- 明細查詢頁
- 專案列表查詢
- 支出逐筆編輯
- DB / API / persistence 設計

---

# 2. 時間主體

營收概況 v1 的時間主體採：

> **以月份為預設主體**

並支援以下兩種延伸視角：

1. 年份統計
2. 自訂統計區間

正式語意：
- 預設先看某個月份的營收概況
- 可切換成某一整年的統計
- 也可選擇自訂起訖期間查看指定區間的營收概況

---

# 3. 頁面結構

營收概況 v1 頁面固定由兩大段構成：

1. 時間範圍控制區
2. 核心摘要區

正式原則：
- 先決定統計範圍
- 再查看該範圍下的核心營運結果

---

# 4. 時間範圍控制區規格

時間範圍控制區由兩部分構成：

## 4.1 統計模式切換
保留三種模式：
1. 月份
2. 年份
3. 區間

## 4.2 對應時間選擇器
- 月份模式 → 選擇月份
- 年份模式 → 選擇年份
- 區間模式 → 選擇起訖區間

正式原則：
> 先選統計模式，再用對應時間選擇器決定實際統計範圍。

---

# 5. 核心摘要區規格

核心摘要區採：

> **4 張摘要卡**

卡片由左到右保留以下內容：

1. 已結案總收入
2. 已結案總成本
3. 營運支出
4. 利潤總計

正式原則：
- 前三張卡為構成值
- 最後一張卡為結果值

---

# 6. 四張摘要卡定義

## 6.1 已結案總收入
定義：
- 該時間範圍內，已結案專案的總收入

## 6.2 已結案總成本
定義：
- 該時間範圍內，已結案專案的總成本

## 6.3 營運支出
定義：
- 該時間範圍內的營運支出總額

模組對齊語意：
> **營運支出 = 管銷成本**

也就是：
- 本卡片承接 `管銷成本` 模組的統計結果

## 6.4 利潤總計
定義：
- 該時間範圍下最終利潤結果

公式：
> **利潤總計 = 已結案總收入 - 已結案總成本 - 營運支出**

正式語意：
- 前三張卡是構成利潤的組成值
- `利潤總計` 是營收概況 v1 的最終結果值

---

# 7. v1 範圍限制

營收概況 v1 先只做：
- 時間範圍控制區
- 4 張摘要卡

v1 不額外展開：
- 明細區
- 專案列表
- 來源拆解表
- 支出 / 收入鑽取明細

正式原則：
> v1 先作為 summary layer，不作為資料查詢頁。

---

# 8. 一句話定稿

> `營收概況` v1 的正式定位，是帳務中心中用來查看某個時間範圍下收入、成本、營運支出與最終利潤的營運摘要區；本區以月份為預設主體，並支援年份統計與自訂區間，頁面先由時間範圍控制區決定統計範圍，再以 4 張摘要卡呈現 `已結案總收入`、`已結案總成本`、`營運支出（= 管銷成本）` 與 `利潤總計`，其中利潤總計的公式為：`已結案總收入 - 已結案總成本 - 營運支出`；v1 先不展開明細區，僅作為 summary layer。
