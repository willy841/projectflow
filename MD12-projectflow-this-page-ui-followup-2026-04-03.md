# 4. 後續待補強 / 待討論主題

下次回來討論時，優先從這些題目繼續：

## 4.1 卡片首屏保留資訊
待討論：
- 首屏現在是否仍偏擠
- 哪些資訊應留在收合狀態
- 哪些資訊應退回展開層
- 哪些資訊如果拿掉，會導致辨識力不夠

這一題的核心不是加欄位，而是：

> **在不新增欄位前提下，首屏要留下哪些現有資訊才最剛好。**

## 4.2 CTA 主次順序是否完全正確
待討論：
- `新增回覆 / 查看回覆 / 查看主卡資訊` 的主次排序是否已經對
- 設計 / 備品 / 廠商三條線，是否適合同一套 CTA 優先順序
- 是否要依卡片當前狀態改變主 CTA

核心問題：

> **這一頁最常用、最該被優先點擊的動作，到底應該怎麼排序。**

## 4.3 展開層閱讀節奏
待討論：
- 打開一張卡後，內容由上到下的順序是否自然
- 母卡資訊、補充資訊、回覆列表、整理入口的節奏是否需要再調
- 是否有某些資訊塊其實應該更後面或更前面

核心問題：

> **使用者點開後，視線是否能不費力地往下走。**

## 4.4 整理層與任務卡的層次關係
待討論：
- 使用者是否能直覺分辨：
  - 上面是任務 / 回覆
  - 下面是整理 / 輸出
- 兩者目前是否看起來過於接近，像同一層內容

核心問題：

> **這頁是否已經清楚分成「任務處理層」與「文件輸出層」。**

## 4.5 狀態語言一致性
待討論：
- badge 命名是否一致
- active 提示文案是否一致
- 文件狀態、回覆狀態、操作狀態的語氣是否統一
- 視覺顏色與命名是否有混亂感

核心問題：

> **功能可能已通，但語言與視覺是否像同一套系統。**

## 4.6 設計 / 備品 / 廠商三線視覺一致性
待討論：
- 卡片密度是否一致
- 按鈕節奏是否一致
- 標題 / 摘要 / badge / CTA 區是否一致
- 是否仍有某一條線特別擠、特別鬆、特別不像同系統

核心問題：

> **三條線是否真的已經收成同一頁，而不是三套 UI 拼在一起。**
