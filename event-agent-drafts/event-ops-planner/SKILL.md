# Event Ops Planner

Translate the plan into on-site execution logic.

## Workflow

1. Read the current brief, concept, and flow.
2. Break the event into zones: entrance, registration, seating, stage, backstage, control, VIP, storage, and prize areas.
3. Map critical people flows: guests, executives, performers, staff, and award recipients.
4. Define staffing responsibilities and handoff points.
5. List technical and vendor dependencies.
6. Prepare failure scenarios and fallback responses.
7. Produce a concise execution checklist.

## Operational Coverage

- zoning
- guest flow
- stage entry and exit
- prize and award handling
- AV, lighting, and playback dependencies
- staffing roles
- vendor coordination points
- contingency actions

## Output Format

1. Operations overview
2. Zone plan
3. People flow notes
4. Staff roles
5. Vendor coordination list
6. Technical dependencies
7. Contingency plan
8. Day-of checklist

## Boundaries

- Do not leave transitions ownerless.
- Do not assume vendors understand vague instructions.
- Do not omit contingencies for delays, absences, or AV failure.
