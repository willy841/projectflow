# MD44 - projectflow Accounting Center 管銷成本模組 v1 spec（2026-04-08）

> 目的：定義 `projectflow` 下游階段中，Accounting Center / 帳務中心之下 `管銷成本` 模組的 v1 正式規格。
>
> 本檔承接：
> - `MD42-projectflow-accounting-center-personnel-input-area-v1-spec-2026-04-08.md`
> - `MD43-projectflow-accounting-center-personnel-record-area-v1-spec-2026-04-08.md`
>
> 本檔目前先不碰 DB、API、persistence，只定義前端功能架構、頁面定位、主結構、子區塊邊界與模組對齊語意。

---

# 1. 模組定位

`管銷成本` 模組的正式定位為：

> **記錄所有實際開銷花費的營運支出模組**

本模組承接：
- 非專案別支出
- 固定週期支出
- 日常營運性質支出

本模組不是：
- 專案成本明細頁
- 廠商付款管理頁
- Closeout archive

---

# 2. 與營收概況的關係

正式對齊語意：

> **管銷成本 = 營運支出**

也就是：
- 在帳務中心中，營收概況區塊內的 `營運支出`
- 正式由 `管銷成本` 模組承接

---

# 3. 模組主體

管銷成本模組 v1 採：

> **以月份為主體**

正式原則：
- 先進入某個月份
- 再查看該月份下的人事 / 庶務 / 其他支出
- 月份為第一層主體，類型為第二層分類

---

# 4. 模組分類

管銷成本 v1 先分三大類：

1. 人事
2. 庶務
3. 其他

---

# 5. 某月份頁面結構

管銷成本某月份頁面固定由上到下為：

1. 月份摘要區
2. 人事區
3. 庶務區
4. 其他區

正式原則：
- 先看該月份總體營運支出概況
- 再往下看三大分類內容

---

# 6. 月份摘要區規格

月份摘要區保留以下 4 個值：

1. 人事成本小計
2. 庶務成本小計
3. 其他成本小計
4. 當月營運支出總計

正式語意：
- 人事成本小計：該月份人事支出總額
- 庶務成本小計：該月份庶務支出總額
- 其他成本小計：該月份其他支出總額
- 當月營運支出總計：該月份整體營運支出總額

---

# 7. 人事區定位

人事區用來承接當月的人事成本管理。

人事區由兩部分構成：

1. 人事輸入區
2. 人事記錄區

## 7.1 人事輸入區
- 以員工為主體輸入人事資料
- 將資料送到指定薪資月份
- 正職與兼職分流編輯

正式承接文件：
- `MD42-projectflow-accounting-center-personnel-input-area-v1-spec-2026-04-08.md`

## 7.2 人事記錄區
- 以月份為主體查看已送出的人事紀錄
- 正職與兼職分開顯示
- 單筆詳情以唯讀抽屜查看

正式承接文件：
- `MD43-projectflow-accounting-center-personnel-record-area-v1-spec-2026-04-08.md`

---

# 8. 庶務區定位

庶務區用來承接當月庶務類營運支出。

後續應再拆成：
1. 庶務輸入區
2. 庶務記錄區

目前狀態：
> **子 spec 尚未完成**

正式原則：
- 先在母 spec 中保留庶務區定位
- 後續再獨立完成庶務輸入區 / 記錄區規格

---

# 9. 其他區定位

其他區用來承接當月其他類營運支出。

後續應再拆成：
1. 其他輸入區
2. 其他記錄區

目前狀態：
> **子 spec 尚未完成**

正式原則：
- 先在母 spec 中保留其他區定位
- 後續再獨立完成其他輸入區 / 記錄區規格

---

# 10. 一句話定稿

> `管銷成本` 模組 v1 的正式定位，是帳務中心中承接所有實際開銷花費的營運支出模組；本模組採「以月份為主體」的結構，先看月份摘要，再往下看人事、庶務、其他三大分類，其中人事區已正式拆成 `人事輸入區` 與 `人事記錄區` 並由既有子 spec 承接，而庶務區與其他區目前先保留模組定位，待後續各自完成輸入區 / 記錄區規格；在模組對齊語意上，`管銷成本 = 營運支出`。
