# 3. Financial Detail 的區塊級對帳規則

## 3.1 對帳不是專案級單一按鈕，而是區塊級動作

Financial Detail 中的「確認對帳完成」不應只有一顆全域按鈕，而應拆成三個區塊各自獨立的動作。

區塊如下：
- 設計
- 備品
- 廠商

## 3.2 按鈕放置位置

「確認對帳完成」按鈕應分別放在：
- 設計卡
- 備品卡
- 廠商卡

每張卡都有自己的：
- 狀態顯示
- 對帳完成動作

## 3.3 對帳狀態定義

每個區塊各自有自己的對帳狀態：
- 未對帳
- 已對帳

語意：
> 某一區塊被標記為「已對帳」，代表該區塊的實際金額 / 成本內容已核對完成，正式成立。

## 3.4 對帳完成後的效力

某區塊一旦已對帳，代表：
1. 該區塊金額正式成立
2. 該區塊對應的廠商應付款資訊，可連動到廠商資料中的「未出款」
3. 後續結案條件會將此區塊視為已完成

重要規則：
> 不是先有廠商未出款，再對帳；而是先完成對帳，該金額才成立，才會連動到廠商未出款。

# 4. 收款狀態與金額規則

## 4.1 收款不能只有 yes/no 狀態

Financial Detail 內的收款資訊不能只顯示：
- 已收定金 / 未收定金
- 已收尾款 / 未收尾款

還必須有對應的金額欄位。

## 4.2 定金資料

定金需至少包含：
1. 是否已收定金
2. 定金已收金額

語意：
- 「是否已收定金」表示定金狀態
- 「定金已收金額」表示實際已收多少錢

## 4.3 尾款資料

尾款需至少包含：
1. 是否已收尾款
2. 尾款剩餘金額

語意：
- 「是否已收尾款」表示尾款狀態
- 「尾款剩餘金額」表示目前還有多少尾款尚未收回

## 4.4 收款成立條件

### 定金已收成立條件
應至少滿足：
- 已標記為已收定金
- 定金已收金額已填，且大於 0

### 尾款已收成立條件
應至少滿足：
- 已標記為已收尾款
- 尾款剩餘金額為 0

# 5. 未結案 / 已結案規則

## 5.1 結案不是一般 badge，而是正式流程節點

「未結案 / 已結案」不能只是 UI 標籤，而必須代表正式流程狀態。

## 5.2 結案條件

專案只有在以下條件全部成立後，才可結案：

### 對帳條件
1. 設計已對帳
2. 備品已對帳
3. 廠商已對帳

### 收款條件
4. 定金已收，且已填定金已收金額
5. 尾款已收，且尾款剩餘金額為 0

總結：
> 三區塊皆已對帳 + 客戶款項皆已收完，才可結案。

## 5.3 結案按鈕位置

結案按鈕應放在：
> 專案總覽區塊（Financial Detail 的第一個專案總覽區塊）

## 5.4 結案按鈕啟用規則

- 未滿足結案條件前：按鈕 disabled
- 滿足條件後：按鈕可點選

## 5.5 結案動作

一旦使用者按下「結案」：
1. 系統將專案狀態標記為 `已結案`
2. 系統將專案資料移入「結案紀錄」

## 5.6 結案後資料承接

結案不是單純前端顯示切換，而是：
> 正式把資料移到結案紀錄中承接。

也就是：
- 結案紀錄是正式承接區
- 不只是某個 badge 的別名

# 6. 專案總覽區塊規則

## 6.1 保留內容

專案總覽區塊應承接：
- 對帳狀態整體摘要
- 結案狀態
- 定金 / 尾款收款資訊
- 結案按鈕

## 6.2 移除內容

專案總覽區塊中的：
> 專案代碼卡片不需要，應移除。

# 7. 帳務中心與首頁摘要的關係

## 7.1 帳務中心是正式帳務主體

本月已收款 / 未收款等正式帳務資料，應以「帳務中心」為主體承接。

帳務中心應承接的內容包括：
- 本月已收款
- 本月未收款
- 收款狀態
- 收款金額
- 專案級帳務明細

## 7.2 首頁不是帳務主體

首頁不應自成一套帳務資料邏輯。
首頁的角色應是：
- 顯示摘要
- 作為入口
- 連結到帳務中心

正式規則：
> 首頁只做帳務摘要入口，正式本月已收 / 未收資料應歸屬於帳務中心。

# 9. 驗收標準

## 9.1 對帳驗收
- 設計 / 備品 / 廠商三張卡可各自獨立標記對帳完成
- 各區塊狀態彼此獨立
- 對帳完成後，該區塊才視為正式成立

## 9.2 收款驗收
- 可記錄定金已收與金額
- 可記錄尾款已收與剩餘金額
- 收款欄位不只是 badge，而是有實際數值

## 9.3 結案驗收
- 未達條件前，結案按鈕不可按
- 達成下列全部條件後才可按：
  - 設計已對帳
  - 備品已對帳
  - 廠商已對帳
  - 定金已收且有金額
  - 尾款已收且剩餘為 0
- 按下結案後，資料可進入結案紀錄

## 9.4 帳務中心 / 首頁驗收
- 本月已收 / 未收款正式歸屬帳務中心
- 首頁僅作摘要入口，不另算一套財務資料
