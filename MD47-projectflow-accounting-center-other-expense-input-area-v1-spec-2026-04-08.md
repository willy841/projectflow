# MD47 - projectflow Accounting Center 其他輸入區 v1 spec（2026-04-08）

> 目的：定義 `projectflow` 下游階段中，Accounting Center / 帳務中心之下 `管銷成本` 模組的 `其他輸入區` v1 正式規格。
>
> 本檔承接：
> - `MD44-projectflow-accounting-center-operating-expense-module-v1-spec-2026-04-08.md`
>
> 本檔目前先不碰 DB、API、persistence，只定義前端功能架構、頁面定位、欄位規則、主操作與刪除語意。

---

# 1. 模組定位

`其他輸入區` 的正式定位為：

> **承接無法歸入人事或庶務，但仍屬於營運支出的其他單筆支出輸入區**

本區只處理：
- 新增單筆其他支出
- 編輯單筆其他支出
- 刪除單筆其他支出

本區不處理：
- 其他記錄查看
- DB / API / persistence 設計

---

# 2. 使用情境

使用者進入其他輸入區後，先在入口頁看到目前其他支出列表，並可透過 `新增` 建立一筆新的其他支出。

若需要修改既有支出，則透過列表中的 `編輯` 以 modal 方式修改。

若需要移除單筆支出，系統需在刪除前先跳出確認提醒，以避免誤刪。

---

# 3. 主體與入口結構

其他輸入區 v1 採：

> **以單筆支出為主體**

入口結構採：

> **列表 + 新增按鈕**

正式原則：
- 先看到支出列表
- 再透過新增 / 編輯 / 刪除處理單筆其他支出

---

# 4. 入口列表規格

入口列表每列保留以下欄位：

1. 項目名稱
2. 金額
3. 備註
4. 編輯

正式語意：
- 項目名稱：辨識該筆其他支出內容
- 金額：顯示該筆支出金額
- 備註：顯示該筆支出的補充說明
- 編輯：進入該筆支出的編輯 modal

---

# 5. 新增支出規格

## 5.1 互動方式

新增支出採：

> **modal**

正式原則：
- 不跳新頁
- 在當前其他輸入區上下文中完成新增

## 5.2 modal 欄位

新增支出 modal v1 保留以下欄位：

1. 項目名稱
2. 金額
3. 備註

---

# 6. 編輯支出規格

## 6.1 互動方式

列表中的 `編輯` 採：

> **modal**

正式原則：
- 與新增支出保持一致
- 不另開獨立頁面

## 6.2 modal 結構

編輯支出 modal 與新增支出採相同欄位結構：

1. 項目名稱
2. 金額
3. 備註

---

# 7. 刪除支出規格

其他輸入區 v1 支援刪除單筆支出。

## 7.1 刪除語意
- 刪除支出 = 將該筆其他支出自輸入區列表中移除

## 7.2 防誤刪規則
- 使用者執行刪除前，系統必須先跳出確認提醒
- 經再次確認後，才可完成刪除

正式原則：
> 其他輸入區支援單筆支出刪除，但必須有防誤刪機制。

---

# 8. 一句話定稿

> `其他輸入區` v1 採「以單筆支出為主體」的輸入結構，承接所有無法歸入人事或庶務、但仍屬於營運支出的其他單筆支出；入口頁先顯示支出列表，並透過新增 / 編輯 modal 處理單筆資料，欄位保留項目名稱、金額、備註；單筆支出可刪除，但必須有防誤刪提醒。
