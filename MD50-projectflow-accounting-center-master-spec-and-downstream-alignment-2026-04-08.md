# 2. Accounting Center 目前正式主結構

`Accounting Center` 目前正式維持三大主區塊：

1. **執行中專案**
2. **管銷成本**
3. **營收概況**

## 2.1 執行中專案
已知正式語意：
- 承接進行中專案的帳務管理
- 已定內容包括：
  - 總金額
  - 已收款
  - 未收款
  - 款項追蹤表

本輪狀態：
- 已完成主結構與款項追蹤表核心規則
- 尚未整理成最終母 spec 單檔

## 2.2 管銷成本
正式語意：
- 記錄所有實際開銷花費
- 承接非專案別、固定週期或日常營運性質的營運支出

本輪狀態：
- 已完成母 spec `MD44`
- 已完成人事、庶務、其他三條子線主要規格

## 2.3 營收概況
正式語意：
- 查看某個時間範圍下的收入、成本、營運支出與最終利潤摘要

本輪狀態：
- 已完成 `MD49`
- v1 先為 summary layer，不做明細層

# 3. 管銷成本主線總整理

## 3.1 模組定位
`管銷成本` 的正式定位：

> **記錄所有實際開銷花費的營運支出模組**

並且已正式定義：

> **管銷成本 = 營運支出**

這條是後續與 `營收概況` 對齊的核心語意，不可再模糊寫成「來源之一」。

## 3.2 管銷成本主體
`管銷成本` 模組採：

> **以月份為主體**

也就是：
- 月份為第一層主體
- 人事 / 庶務 / 其他為第二層分類

## 3.3 管銷成本某月份頁面固定結構
1. 月份摘要區
2. 人事區
3. 庶務區
4. 其他區

## 3.4 月份摘要區固定保留 4 個值
1. 人事成本小計
2. 庶務成本小計
3. 其他成本小計
4. 當月營運支出總計

# 4. 人事主線總整理

## 4.1 人事區定位
人事區正式由兩部分構成：
1. **人事輸入區**
2. **人事記錄區**

---

## 4.2 人事輸入區（MD42）
### 入口頁定位
- 員工編輯入口
- 員工名單管理入口
- 不承接月份管理
- 不承接歷史記錄查詢

### 入口頁內容
- 正職人數卡片
- 兼職人數卡片
- 員工列表（姓名 / 類型 / 編輯）
- 新增員工
- 刪除員工

### 入口頁重要規則
- 點正職卡片 → 只顯示正職員工
- 點兼職卡片 → 只顯示兼職員工
- 新增員工採 modal，欄位只有：姓名、類型
- 刪除員工採真的刪除，但必須有防誤刪

### 正職編輯頁
固定區塊：
1. 基本資訊區
2. 薪資日期區
3. 應支付項目區
4. 加班計算區
5. 應扣項目區
6. 單位負擔區
7. 頁面底部操作區

關鍵規則：
- 薪資日期只承接年 / 月
- 應支付明細、其他扣項、其他單位負擔皆採明細型
- 加班拆成三組算式
- 應支合計包含加班費
- 人事成本總支出 = 應支合計 + 單位負擔合計
- 底部按鈕：取消 / 暫存 / 送出
- 送出時若同員工同月份已有既有紀錄，需先確認是否覆蓋

### 兼職編輯頁
固定區塊：
1. 基本資訊區
2. 薪資日期區
3. 兼職薪資計算區
4. 頁面底部操作區

關鍵規則：
- 欄位：本月工作時數 / 每小時薪資金額 / 本月應支金額
- 不承接正職型複雜結構
- 同樣有暫存 / 送出 / 覆蓋提醒邏輯

---

## 4.3 人事記錄區（MD43）
### 主體
- 以月份為主體
- 預設顯示最近月份
- 可切換其他月份

### 某月份頁面固定結構
1. 月份摘要區
2. 正職記錄區
3. 兼職記錄區

### 月份摘要區固定保留 5 個值
1. 正職人數
2. 兼職人數
3. 正職成本小計
4. 兼職成本小計
5. 人事總成本

### 正職記錄列表
欄位：
- 姓名
- 應支合計
- 應扣合計
- 單位負擔合計
- 人事成本總支出
- 查看詳情

### 兼職記錄列表
欄位：
- 姓名
- 本月工作時數
- 每小時薪資金額
- 本月應支金額
- 查看詳情

### 查看詳情方式
- 採側邊抽屜
- 正職與兼職抽屜分流
- 抽屜沿用輸入頁結構做唯讀版

---

## 5.2 庶務輸入區（MD45）
### 主體
- 以單筆支出為主體

### 入口結構
- 列表 + 主操作

### 列表欄位
- 項目名稱
- 分類
- 金額
- 編輯

### 主操作
- 新增支出
- 管理分類

### 新增 / 編輯方式
- 皆採 modal
- 欄位：項目名稱 / 分類 / 金額 / 備註

### 分類規則
- 單筆支出時，分類從既有清單中選擇
- 不採自由輸入
- 分類清單本身可管理，支援新增 / 刪除
- 分類管理入口放在庶務輸入區入口頁

### 刪除規則
- 支援單筆支出刪除
- 必須有防誤刪提醒

# 7. 營收概況總整理（MD49）

## 7.1 模組定位
`營收概況` 是：

> **用來查看某個時間範圍下，收入、成本、營運支出與最終利潤的營運摘要區**

## 7.2 時間主體
- 以月份為預設主體
- 可切換為年份統計
- 可選擇自訂區間統計

## 7.3 頁面固定結構
1. 時間範圍控制區
2. 核心摘要區

## 7.4 時間範圍控制區
- 統計模式切換：月份 / 年份 / 區間
- 對應時間選擇器

## 7.5 核心摘要區
採 **4 張摘要卡**：
1. 已結案總收入
2. 已結案總成本
3. 營運支出
4. 利潤總計

### 利潤總計公式
> **利潤總計 = 已結案總收入 - 已結案總成本 - 營運支出**

### 重要對齊語意
- 營運支出 = 管銷成本
- v1 先不展開明細區
- v1 只作為 summary layer

# 8. 與 Closeout / Vendor Data 的目前邊界理解

## 8.1 與 Closeout 的對齊
目前已可先穩定理解為：
- `已結案總收入`
- `已結案總成本`

這兩個值的產品語意，應優先對齊 `Closeout` 已完成專案的最終結果。

也就是：
- `Closeout` 承接已結案專案的留存與查閱
- `營收概況` 則承接這些已結案結果的摘要統計

## 8.2 與 Vendor Data 的邊界
目前已定理解：
- `Vendor Data` 不承接帳務中心
- Vendor Data 處理 vendor payable / 未付款 / 合作紀錄
- `Accounting Center` 處理 client 收款、營運支出與營收概況摘要

正式原則：
> 不要再把 Vendor Data 與 Accounting Center 混成同一條線。
